# 物流小车底盘控制板 (V1.0) 硬件接口与全引脚映射规约

## 1. 核心架构与微控制器 (MCU) 规范
本控制板采用双 MCU 架构，核心主控为两片 **STM32F103C8T6**（采用标准 40-Pin BluePill 核心板封装与引脚顺序）。
- **主机 (Master)**：负责上层逻辑、上位机通讯、传感器(IMU/OLED)采集与舵机(PWM)控制。
- **副机 (Slave)**：作为底层驱动协处理器，专职负责将主机的运动指令转换为 RS485 总线协议，与四个底盘电机进行实时双向通讯。
- **全局网络标识极度重要提示**：本硬件系统中，所有地线回路均严格标识为 **`G`**（无 `GND` 标识）。电源正极根据回路不同标识为 **`V+`**（或具体的 12.6V、5V、3.3V）。硬件接线与查线时必须以此为准。

## 3. 核心通讯链路拓扑与数据流

### 3.1 跨芯片指令链路 (主机 USART2 $\leftrightarrow$ 副机 USART2)
实现控制逻辑与底层电机驱动的物理隔离：
- 主机发送：主机 `PA2` (USART2_TX) $\rightarrow$ 副机 `PA3` (USART2_RX)
- 副机反馈：副机 `PA2` (USART2_TX) $\rightarrow$ 主机 `PA3` (USART2_RX)

### 3.2 底盘 RS485 动力总线 (副机 USART3 $\leftrightarrow$ MAX3485 $\leftrightarrow$ 底盘电机)
副机通过 USART3 驱动 MAX3485 芯片进行半双工差分通讯：
- **流向**：副机 `PB10` (TX) $\rightarrow$ MAX3485 $\rightarrow$ 总线 `BUS_S_A` / `BUS_S_B` $\rightarrow$ M1~M4 并联接口 $\rightarrow$ 底盘轮毂电机。
- **方向控制**：副机使用普通 GPIO `PB1` 控制通讯方向。

## 4. 外部接口物理线序细节

### 4.1 保留位&上位机接口 (H5)
对接外部 ROS 工控机或 PC：
- 1脚: `M_MASTER_TX` (对接上位机 RX，来自主机 `PA9`)
- 2脚: `M_MASTER_RX` (对接上位机 TX，来自主机 `PA10`)
- 电源: 需共地，对接本板的 **`G`**。

### 4.2 四驱底盘电机接口 (M1, M2, M3, M4)
4个 8-Pin 端子，内部完全并联：
- **Pin 1-2**: `12.6V` (`V+`) 主动力电源
- **Pin 4**: `BUS_S_A` (RS485 A线)
- **Pin 5**: `BUS_S_B` (RS485 B线)
- **Pin 7-8**: **`G`** (系统共地)

### 4.3 独立串行拓展接口 (M5)
用于连接独立串口控制的顶层外设（如串口机械臂）：
- **Pin 1-2**: `12.6V` (`V+`) 
- **Pin 4**: `M_MOTORS_RX` (来自主机 `PB11`)
- **Pin 5**: `M_MOTORS_TX` (来自主机 `PB10`)
- **Pin 7-8**: **`G`**

### 4.4 舵机/云台 PWM 接口 (H1, H2, H3)
- H1: **`G`** / `5V` (`V+`) / `M_PWM1` (主机 `PA7`)
- H2: **`G`** / `5V` (`V+`) / `M_PWM2` (主机 `PA6`)
- H3: **`G`** / `5V` (`V+`) / `M_PWM3` (主机 `PA8`)

### 4.5 传感器 I2C 总线复用接口
主机通过一组硬件 I2C（`PB6`=SCL, `PB7`=SDA）挂载不同电压外设，I2C1 与 I2C2 共用同一组 SCL/SDA 引脚：
- **OLED 屏幕 (I2C1)**: 接 `3.3V` (`V+`), **`G`**, `PB6` (SCL), `PB7` (SDA)
- **陀螺仪 IMU (I2C2)**: 接 `5V` (`V+`), **`G`**, `PB6` (SCL), `PB7` (SDA)
