# 智能搬运小车项目文档（chassis 主机 / chassis2 副机）

> 本文档依据当前 `chassis/`（主机固件）与 `chassis2/`（副机固件）源码整理，
> 描述系统架构、硬件接线、通信协议、各功能模块与控制流程，供开发与联调参考。
> 所有多字节字段均为**小端序**（里程计 rpm 字段除外，见 §4.5）。

---

## 1. 系统总览

整车采用 **双 STM32F103C8T6 主从架构**：

```
                ┌────────────────────────────────────────────┐
   上位机/PC ───┤ USART1 (230400, DMA)                        │
   (A5 5A 协议) │                                            │
                │              chassis 主机                   │
                │   - IMU 采集 (JY901B, I2C1, 200Hz)          │
                │   - 塔吊舵机 (物料盘/底座/爪子)             │
                │   - M5 升降步进 (USART3 直连 EMM42)         │
                │   - 协议解析 + 指令分发 + 看门狗            │
                │                                            │
                │ USART2 (115200, DMA) ◄──── 主从双向 ───────┤
                └──────────────────────┬─────────────────────┘
                                       │  A5 5A 协议
                ┌──────────────────────┴─────────────────────┐
                │              chassis2 副机                  │
                │   - 接收主机运动指令 (USART2, 中断)         │
                │   - RS485 (USART3) 驱动 4 个 EMM42 轮电机   │
                │   - 50Hz 轮询轮速 → 里程计上报主机          │
                │   - EMM42 使能心跳 + UART 健康自愈          │
                └─────────────────────────────────────────────┘
```

- **主机**（chassis）：面向上位机，负责 IMU、塔吊机械臂（舵机 + 升降电机）以及把底盘运动指令转发给副机。
- **副机**（chassis2）：面向 4 个麦克纳姆轮电机，通过 RS485 闭环驱动，并周期回传轮速里程计。
- **上位机 ↔ 主机**：USART1，230400 8N1，DMA 收发。
- **主机 ↔ 副机**：USART2，115200 8N1，主机 DMA / 副机中断。
- 两块板各自带 **IWDG 看门狗（≈2.4s）**，长阻塞流程主动喂狗。

## 2. 硬件接线

### 2.1 主机 (chassis)

| 外设            | 接口            | 引脚        | 说明                              |
| --------------- | --------------- | ----------- | --------------------------------- |
| 上位机串口      | USART1          | PA9/PA10    | 230400, DMA 收发                  |
| 主从串口        | USART2          | PA2/PA3     | 115200, DMA, 与副机双向           |
| 升降/M5 电机    | USART3          | PB10/PB11   | 115200, TTL 直连 EMM42 (非 RS485) |
| IMU JY901B      | I2C1            | PB6/PB7     | 200Hz 轮询                        |
| 物料盘舵机      | TIM3_CH2        | PA7         | DS3225, 50Hz PWM                  |
| 塔吊底座舵机    | TIM3_CH1        | PA6         | DS3225, 50Hz PWM                  |
| 爪子舵机        | TIM1_CH1        | PA8         | DS3225, 50Hz PWM（高级定时器）    |
| 状态 LED        | GPIO            | LED_Pin     | 1Hz 心跳                          |

> 升降电机（lift_motor）与 M5 高精度控制层（m5_motor）**物理上是同一台 EMM42（地址 5）**，
> 共享 USART3，主循环串行调用，互不冲突。

### 2.2 副机 (chassis2)

| 外设          | 接口    | 引脚       | 说明                          |
| ------------- | ------- | ---------- | ----------------------------- |
| 主从串口      | USART2  | PA2/PA3    | 115200, 中断接收              |
| RS485 总线    | USART3  | PB10/PB11  | 115200, 4 个 EMM42 轮电机     |
| RS485 方向控制| GPIO    | PB1        | 收发方向切换                  |
| 状态 LED      | GPIO    | LED_Pin    | 1Hz 心跳                      |

**轮电机地址**：FL=1（前左）、FR=2（前右）、RL=3（后左）、RR=4（后右），广播=0。

### 2.3 系统时钟

两板一致：HSE 8MHz → PLL×9 → **SYSCLK 72MHz**，APB1=36MHz（定时器倍频 72MHz）、APB2=72MHz，FLASH 2 等待。

## 3. 串口协议（A5 5A）

### 3.1 帧结构

```
SOF1 | SOF2 | CFG | SEQ | SRC | DST | FUNC | CMD | LEN(2B) | PAYLOAD | CRC16(2B)
```

| 字段    | 长度 | 说明                                    |
| ------- | ---- | --------------------------------------- |
| SOF1    | 1B   | `0xA5`                                  |
| SOF2    | 1B   | `0x5A`                                  |
| CFG     | 1B   | bit7-4 版本; bit1 IS_ACK; bit0 NEED_ACK |
| SEQ     | 1B   | 序号 0~255                              |
| SRC/DST | 1B×2 | 地址（上位机 0x01 / 主机 0x10 / 副机 0x02 / PC 0x00） |
| FUNC    | 1B   | 功能域，见 §3.2                         |
| CMD     | 1B   | 子命令                                  |
| LEN     | 2B   | PAYLOAD 长度（小端）                    |
| PAYLOAD | 可变 | 数据区                                  |
| CRC16   | 2B   | Modbus CRC16（小端）                    |

- **CRC16**：CRC-16/IBM(Modbus)，Poly=0xA001(反射)，Init=0xFFFF，覆盖 `SOF1..PAYLOAD`。校验失败的帧直接丢弃。
- 标准 CFG 取值：普通帧 `0x10`，需 ACK `0x11`，ACK 帧 `0x12`。

### 3.2 功能域 (FUNC)

| FUNC   | 含义        | 处理方                       |
| ------ | ----------- | ---------------------------- |
| `0x01` | 系统/错误   | 主从均可上报 ERR             |
| `0x02` | 底盘运动    | 主机转发 → 副机执行          |
| `0x03` | 舵机/塔吊/M5| 主机本地执行                 |
| `0x04` | 传感器/IMU  | 主机本地执行                 |

### 3.3 底盘命令 (FUNC=0x02)

| CMD    | 名称       | 方向    | PAYLOAD                                     |
| ------ | ---------- | ------- | ------------------------------------------- |
| `0x01` | SET_VEL    | 上→下   | v1..v4 int16 + accel(1B, 可选) = 8/9B       |
| `0x02` | SET_POS    | 上→下   | p1..p4 int32 + rpm(u16) + accel(1B) + abs(1B) = 20B |
| `0x04` | STOP       | 上→下   | 空                                          |
| `0x05` | ENABLE     | 上→下   | enable(1B)                                  |
| `0x06` | SYNC       | 上→下   | 空                                          |
| `0x07` | ODOM_REPORT| 下→上   | 12B，见 §4.5                                |

### 3.4 舵机/塔吊/M5 命令 (FUNC=0x03)

主机 `main.c` 按 CMD 范围分发：**`0x10~0x1F` → M5 升降电机；其余 → 塔吊/舵机**。

塔吊/舵机子命令（`TowerCrane_HandleFrame`）：

| CMD    | 名称            | PAYLOAD                          |
| ------ | --------------- | -------------------------------- |
| `0x01` | SERVO_SET_ANGLE | id(1B) + angle_x10(int16 LE)     |
| `0x02` | PLATE_NEXT      | forward(1B)                      |
| `0x03` | PLATE_GOTO      | slot(1B)                         |
| `0x04` | BASE_SET        | working(1B)                      |
| `0x05` | CLAW_SET        | close(1B)                        |
| `0x06` | LIFT_MOVE       | height_mm_x10(int16 LE)          |
| `0x07` | LIFT_STOP       | 空                               |
| `0x08` | PICK_PLACE      | 空（整套抓放流程）               |
| `0x09` | NEXT_AND_PICK   | 空（转下一格并抓取）             |

M5 升降子命令（`M5_HandleFrame`）：

| CMD    | 名称            | PAYLOAD                          |
| ------ | --------------- | -------------------------------- |
| `0x10` | M5_ENABLE       | enable(1B)                       |
| `0x11` | M5_STOP         | 空                               |
| `0x12` | M5_SPEED        | ccw(1B) + rpm(u16 LE)（连续转速模式）|
| `0x13` | M5_MOVE_DEG     | deg_x100(int32 LE) 相对角度      |
| `0x14` | M5_MOVE_DEG_ABS | deg_x100(int32 LE) 绝对角度      |
| `0x15` | M5_SET_MOVE_RPM | rpm(u16 LE) 设置定位转速（不触发运动）|

> **M5 是角度差控制**：`M5_MoveAngleAbs(deg)` 内部计算 `delta = deg − 当前记账角度`，
> 再换算脉冲下发，软件记账绝对角度。定位命令（0x13/0x14）使用的转速默认 20RPM，
> 可由 `0x15` 在线修改（范围 1~300RPM，内部夹紧）。`0x12` 是连续转速模式，会让电机
> 持续旋转，**不可**用于定位前提速。

### 3.5 传感器命令 (FUNC=0x04)

| CMD    | 名称        | 方向    | PAYLOAD                         |
| ------ | ----------- | ------- | ------------------------------- |
| `0x01` | GET_IMU     | 双向    | 上行请求空；下行 40B（见 §4.4） |

### 3.6 系统/错误 (FUNC=0x01)

| CMD    | 名称        | PAYLOAD                              |
| ------ | ----------- | ------------------------------------ |
| `0x01` | SYS_ERROR   | tick_ms(u32 LE) + ASCII 文本(≤60B)   |

## 4. PAYLOAD 详解

### 4.1 SET_VEL (8/9B)
v1..v4：int16 轮速 RPM（正负表示方向）；accel：可选加速度档位（0=不加减速）。

### 4.2 SET_POS (20B)
p1..p4：int32 目标脉冲（正负表示方向）；rpm：u16 运动速度；accel：u8；absolute：u8（0=相对/1=绝对）。
副机据此对 4 轮分别下发位置命令并 `SyncStart` 同步启动。

### 4.3 SERVO_SET_ANGLE (3B)
id：舵机 ID（0=物料盘 / 1=底座 / 2=爪子）；angle_x10：int16，角度 ×10（即 0.1° 分辨率）。

### 4.4 IMU 上报 (40B)
- `[0..35]`：9 个 float —— ax,ay,az, wx,wy,wz, roll,pitch,yaw（与 `JY901B_Data` 内存对齐，36B 无 padding）。
- `[36..39]`：u32 采样时刻 `tick_ms`（IMU 读取完成时的主机时间戳，便于上位机时间对齐）。
- 主机 **30Hz 主动上报**（SRC=0x10, DST=0x00），同时兼容上位机 GET_IMU 请求即时响应。

### 4.5 ODOM_SPEED 上报 (12B)
副机 **50Hz** 上报 4 轮实时转速，来自 EMM42 `0x35` 回读，单电机超时容错（沿用上一周期缓存）。
每轮 3B：sign(1B, 0=CW/1=CCW) + rpm 高字节 + rpm 低字节，按 FL,FR,RL,RR 顺序。
**注意：rpm 为大端**（EMM42 原生大端透传），与本协议默认小端不同。

## 5. 主机功能模块 (chassis)

### 5.1 主循环节拍

| 周期    | 任务                                    |
| ------- | --------------------------------------- |
| 5ms     | I2C 轮询 JY901B（200Hz）                |
| 33ms    | IMU 主动上报（≈30Hz）                   |
| 1000ms  | LED 心跳 + `stm321alive` 打印           |
| 每轮    | 喂狗、解析上位机/副机数据流             |

- 上位机数据 → 全局解析器 `on_host_frame`（本地执行或转发副机）。
- 副机数据 → 独立解析器 `on_slave_frame`（CRC 校验通过后整帧透传上位机）。

### 5.2 塔吊机械臂 (servo + tower_crane)

3 路 DS3225 舵机，PWM 周期 20ms、脉宽 500~2500µs 对应 0~270°（定时器 1MHz tick，CCR 即脉宽）。

| 部件   | 关键角度（servo.h）                             |
| ------ | ----------------------------------------------- |
| 物料盘 | HOME=142.7°，步进 120°（3 工位，上限放开到 360°）|
| 底座   | HOME=81.6°，WORK=264.0°                          |
| 爪子   | 张开 140°，闭合 80°                             |

**物料盘几何（三角形）**：开机红色（slot 0）正对夹爪；绿色（slot 1）在夹爪初始朝向左侧（+120°）；蓝色（slot 2）在右侧（−120°）。

`tower_crane.c` 提供原子动作（旋转/底座/爪子/升降）与整套 `PickAndPlace` 阻塞流程，
内部统一走 `TC_DelayMs()`，每 50ms 喂一次狗，避免长流程被看门狗复位。

动作时延（tower_crane.h）：物料盘 600ms、底座 700ms、爪子 400ms、升降 1500ms（粗估）。

### 5.3 升降/M5 电机 (lift_motor + m5_motor)

- **同一台 EMM42（地址 5）**，16 细分（3200 脉冲/圈），USART3 直连。
- `lift_motor`：按高度（mm）控制，默认 300RPM；`Lift_GetUart/Lift_TxRaw` 供 M5 层复用总线。
- `m5_motor`：慢速高精度角度控制层，定位默认 20RPM（3 秒/圈），上限 300RPM，加速度档位 200 平滑 S 曲线。

## 6. 副机功能模块 (chassis2)

### 6.1 主循环节拍

| 周期    | 任务                                                |
| ------- | --------------------------------------------------- |
| 每轮    | 处理主机指令（环形缓冲 → 帧状态机）                 |
| 20ms    | 轮询 4 轮转速并上报里程计（50Hz）                   |
| 500ms   | USART2 RX 健康检查（自愈）                          |
| 500ms   | EMM42 使能心跳（广播 enable=1）                     |
| 1000ms  | LED 心跳 + `stm322alive` 打印                       |

### 6.2 EMM42 RS485 驱动

- 4 个闭环步进，校验固定 `0x6B`，命令格式 `addr + cmd + data + 0x6B`。
- 速度模式 `0xF6`、位置模式 `0xFD`、停止 `0xFE`、同步 `0xFF`、读转速 `0x35`、读位置 `0x36`。
- 所有 RS485 操作在主循环串行执行，**天然防总线竞争**。

### 6.3 容错与自愈

1. **里程计超时容错**：单电机读速失败时沿用上一周期缓存值，不中断整轮上报。
2. **UART 自愈**：500ms 检查 USART2 RxState，异常（含 ORE）时仅屏蔽 `USART2_IRQn` 临界区，
   `AbortReceive_IT` 后重启接收、清空残帧，覆盖断电重连/长静默/突发干扰。
3. **使能心跳**：500ms 广播 `enable=1`，应对用户中途断/上电 12V 导致驱动器复位为 disabled，
   最迟 1 秒内自动恢复，无需复位 MCU。

## 7. 典型控制流程

### 7.1 底盘运动（上位机 → 主机 → 副机）

```
上位机 ──SET_VEL(FUNC=0x02,CMD=0x01)──► 主机 on_host_frame
        主机 MComm_SendVelocity ──USART2──► 副机 ProcessFrame
        副机 EMM42_SetWheelSpeeds ──RS485──► 4 轮电机
        副机 50Hz ──ODOM_REPORT(0x07)──► 主机 ──透传──► 上位机
```

### 7.2 取放物料（物料盘 → 工作位，主机本地）

1. 物料盘转到目标物料位置（slot 0/1/2）
2. M5 下降到物料盘取物位置 → 爪子闭合夹取 → M5 复位
3. 底座转到工作位 → M5 下降到放物位 → 爪子张开 → M5 复位
4. 底座复位

> 上位机工具脚本见 `tools/singleaction.py`（物料盘→工作位）与 `tools/singleplace.py`（工作位→物料盘），
> 公共控制库 `tools/chassis_arm.py`。M5 定位速度可通过 `M5_SET_MOVE_RPM(0x15)` 在线调整以缩短流程耗时。

## 8. 构建与烧录

两板均为 PlatformIO + STM32Cube HAL：

```ini
[env:genericSTM32F103C8]
platform  = ststm32
board     = genericSTM32F103C8
framework = stm32cube
build_flags = -D USE_HAL_DRIVER -D STM32F103xB -D HSE_VALUE=8000000U
```

- 主机工程目录 `chassis/`，副机工程目录 `chassis2/`。
- 烧录：`pio run -t upload`（各自工程目录下）。
