# 26AutoGo Transport — 智能搬运小车下位机固件

基于 **双 STM32F103C8T6 主从架构** 的智能搬运小车下位机固件工程。整车由两块单片机协同工作：

- **`master/` 主单片机**：面向上位机，负责协议解析、IMU 采集、塔吊机械臂（舵机 + 升降电机）控制，并把底盘运动指令转发给副机。
- **`slave/` 副单片机**：面向 4 个麦克纳姆轮电机，通过 RS485 闭环驱动 EMM42 步进电机，并周期回传轮速里程计。

> 更完整的系统架构、硬件接线、通信协议与控制流程请参阅 [`docs/项目文档.md`](docs/项目文档.md)。

## 2. 系统架构

```text
                ┌────────────────────────────────────────────┐
   上位机/PC ───┤ USART1 (230400, DMA)                        │
   (A5 5A 协议) │                                            │
                │           master 主单片机                   │
                │   - IMU 采集 (JY901B, I2C1, 200Hz)          │
                │   - 塔吊舵机 (物料盘/底座/爪子)             │
                │   - M5 升降步进 (USART3 直连 EMM42)         │
                │   - 协议解析 + 指令分发 + 看门狗            │
                │                                            │
                │ USART2 (115200, DMA) ◄──── 主从双向 ───────┤
                └──────────────────────┬─────────────────────┘
                                       │  A5 5A 协议
                ┌──────────────────────┴─────────────────────┐
                │           slave 副单片机                    │
                │   - 接收主机运动指令 (USART2, 中断)         │
                │   - RS485 (USART3) 驱动 4 个 EMM42 轮电机   │
                │   - 50Hz 轮询轮速 → 里程计上报主机          │
                │   - EMM42 使能心跳 + UART 健康自愈          │
                └─────────────────────────────────────────────┘
```

- **上位机 ↔ 主单片机**：USART1，230400 8N1，DMA 收发。
- **主单片机 ↔ 副单片机**：USART2，115200 8N1（主机 DMA / 副机中断）。
- 两块板各自带 **IWDG 看门狗（≈2.4s）**，长阻塞流程主动喂狗。
- 所有多字节字段均为 **小端序**（里程计 rpm 字段除外，为大端，详见协议文档）。

## 3. 硬件接线

### 3.1 主单片机 (master)

| 外设         | 接口     | 引脚      | 说明                              |
| ------------ | -------- | --------- | --------------------------------- |
| 上位机串口   | USART1   | PA9/PA10  | 230400, DMA 收发                  |
| 主从串口     | USART2   | PA2/PA3   | 115200, DMA, 与副机双向           |
| 升降/M5 电机 | USART3   | PB10/PB11 | 115200, TTL 直连 EMM42 (非 RS485) |
| IMU JY901B   | I2C1     | PB6/PB7   | 200Hz 轮询                        |
| 物料盘舵机   | TIM3_CH2 | PA7       | DS3225, 50Hz PWM                  |
| 塔吊底座舵机 | TIM3_CH1 | PA6       | DS3225, 50Hz PWM                  |
| 爪子舵机     | TIM1_CH1 | PA8       | DS3225, 50Hz PWM（高级定时器）    |
| 状态 LED     | GPIO     | LED_Pin   | 1Hz 心跳                          |

> 升降电机（lift_motor）与 M5 高精度控制层（m5_motor）**物理上是同一台 EMM42（地址 5）**，共享 USART3，主循环串行调用，互不冲突。

### 3.2 副单片机 (slave)

| 外设           | 接口   | 引脚      | 说明                      |
| -------------- | ------ | --------- | ------------------------- |
| 主从串口       | USART2 | PA2/PA3   | 115200, 中断接收          |
| RS485 总线     | USART3 | PB10/PB11 | 115200, 4 个 EMM42 轮电机 |
| RS485 方向控制 | GPIO   | PB1       | 收发方向切换              |
| 状态 LED       | GPIO   | LED_Pin   | 1Hz 心跳                  |

**轮电机地址**：FL=1（前左）、FR=2（前右）、RL=3（后左）、RR=4（后右），广播=0。

### 3.3 系统时钟

两板一致：HSE 8MHz → PLL×9 → **SYSCLK 72MHz**，APB1=36MHz（定时器倍频 72MHz）、APB2=72MHz，FLASH 2 等待。

## 4. 通信协议（A5 5A）

帧结构：

```text
SOF1 | SOF2 | CFG | SEQ | SRC | DST | FUNC | CMD | LEN(2B) | PAYLOAD | CRC16(2B)
```

- **SOF1/SOF2**：固定 `0xA5 0x5A`
- **地址**：上位机 `0x01` / 主机 `0x10` / 副机 `0x02` / PC `0x00`
- **CRC16**：CRC-16/IBM(Modbus)，Poly=0xA001(反射)，Init=0xFFFF，覆盖 `SOF1..PAYLOAD`，校验失败的帧直接丢弃。

功能域 (FUNC)：

| FUNC   | 含义         | 处理方           |
| ------ | ------------ | ---------------- |
| `0x01` | 系统/错误    | 主从均可上报 ERR |
| `0x02` | 底盘运动     | 主机转发→副机执行 |
| `0x03` | 舵机/塔吊/M5 | 主机本地执行     |
| `0x04` | 传感器/IMU   | 主机本地执行     |

> 完整命令字、PAYLOAD 字段定义、ACK 机制见 [`docs/协议文档.md`](docs/协议文档.md) 与 [`docs/项目文档.md`](docs/项目文档.md)。

## 5. 主要功能模块

### 5.1 主单片机 (master)

- **协议解析与分发**：上位机数据走全局解析器 `on_host_frame`（本地执行或转发副机）；副机数据走独立解析器，CRC 校验通过后整帧透传上位机。
- **IMU 采集**：JY901B 经 I2C1 200Hz 轮询，30Hz 主动上报姿态（带采样时间戳）。
- **塔吊机械臂**：3 路 DS3225 舵机（物料盘 / 底座 / 爪子），提供原子动作与整套 `PickAndPlace` 抓放流程。
- **升降/M5 电机**：同一台 EMM42（地址 5），`lift_motor` 按高度控制，`m5_motor` 慢速高精度角度控制。

### 5.2 副单片机 (slave)

- **运动指令执行**：接收主机 `SET_VEL` / `SET_POS` / `STOP` / `ENABLE` / `SYNC`，经 RS485 驱动 4 个 EMM42 闭环步进电机。
- **里程计上报**：50Hz 轮询 4 轮转速（EMM42 `0x35` 回读）并上报主机，单电机超时容错。
- **容错与自愈**：里程计超时容错、USART2 RX 健康自愈、EMM42 使能心跳（500ms 广播 enable，应对驱动器断电复位）。

## 6. 构建与烧录

两板均为 **PlatformIO + STM32Cube HAL**，配置如下：

```ini
[env:genericSTM32F103C8]
platform  = ststm32
board     = genericSTM32F103C8
framework = stm32cube
build_flags = -D USE_HAL_DRIVER -D STM32F103xB -D HSE_VALUE=8000000U
```

分别在各自工程目录下编译烧录：

```bash
# 主单片机
cd master && pio run -t upload

# 副单片机
cd slave && pio run -t upload
```
